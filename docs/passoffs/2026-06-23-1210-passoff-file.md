## Key decisions

- Live data is fetched directly in-browser (no backend/proxy) because the feed
  is public and sends `Access-Control-Allow-Origin: *`.
- The structural backbone (group draw, R32/KO bracket, third-place annex, FIFA
  ranks) stays embedded in the HTML data island; only group scores are overlaid
  from the API, matched by `(group, unordered team pair)` with score orientation.
- Standings count finished games + user what-ifs only; live in-progress scores
  show in the fixture row but are not half-counted (matches the feed's table).
- The clinched-winner lock follows real results only (`official()`), never edits
  or live games.
- App moved to root `index.html` (+ `.nojekyll`) so Pages serves it at the clean
  root URL with no redirect.
